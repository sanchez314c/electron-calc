### Contributing Content

1. **Fork the repository**
2. **Create a documentation branch**: `git checkout -b docs/update-guide-name`
3. **Make your changes**
4. **Follow documentation style** (consistent formatting, clear language)
5. **Submit a pull request**

### Documentation Style Guidelines

- Use **clear, simple language** (avoid jargon when possible)
- Include **code examples** for technical documentation
- Add **screenshots** for user-facing guides
- Follow **consistent formatting** (Markdown standards)
- Include **table of contents** for long documents
- **Cross-reference** related documents
